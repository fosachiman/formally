Formally

Responsive, Accessible, Classy Form Elements

Formally is a front-end module that aims to allow for easy creation of form elements that are screen responsive and are accessible to those with vision impairments.

Take for example, the input generated below meant to capture a user's first name.

And the simple line of code written to create the text box as well as the label and input guide associated with it:

      
        <input class="f-required" formally="First Name:" id="firstName" type="text">

Let's go through each of the attributes within the form element to describe what each of them is doing behind the scenes.

The class "f-required" can be used when you would like to make a field required. In this case, the requirement is that this field is filled in with any character (since a first name can really be anything). While including this class is optional, it does a few things for us. It adds the native "required" attribute to the element, and the element will now have instant verification shown via the box being highlighted green for a successful input and red for an unsuccessful one. Formally verification is friendly in that a red box outline will only be shown in the case of an unsuccessful entry after the user has left the focus of the input box. If an entry was never made, the input will still stand by for one before warning a user of an unsuccessful entry. It also will load the input guide beneath the text box letting the user know that input is required.

The "formally" attribute does a couple of things for us as well. It instantiates input boxes as Formally elements, so they will receive the default Formally styling. The value of the attribute creates and sets the name of the label, a vital step in making sure our forms are accessible to the masses.

The "id" attribute allows Formally to attribute the label correctly, another vital accessibility feature.

There are two additional classes that can be used to take advantage of Formally's instant form verification:

      
        class="f-email"
        class="f-password"

f-email is used to check for a valid email address based on RFC822. f-password validates only on a password containing at least one uppercase letter, a number, and a special character ($@$!%*#?&). 

      
        <input class="f-password" formally="password" id="password" type="password">

The final attribute type is the HTML type attribute for all input elements. Let's take a look at some other types...

Foods I Enjoy

Radio Buttons:

      
        <input id="ranch" formally="Cool Ranch" type="radio" name="doritos" value="ranch" checked>
        <input id="nacho" formally="Nacho Cheese" type="radio" name="doritos" value="nacho-cheese">

Checkboxes:

      
        <input id="tacos" formally="Tacos" type="checkbox" name="tacos" value="tacos">
        <input id="burritos" formally="Burritos" type="checkbox" name="burritos" value="burritos">

In the case of radio buttons and checkboxes, the label will automatically be appended after the input element.

Buttons!

An optional formally attribute can be added to a select box to provide it with a label describing its contents. It will default to formatting much like the form labels rather than the header used for the button. Both select boxes and buttons (as well as text-areas) follow the same formula as above.
