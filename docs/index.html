Formally

Responsive, Accessible, Classy Form Elements

Formally is a front-end module that aims to allow for easy creation of classy form elements that are screen responsive and are accessible to those with vision impairments.

Take for example, the input box generated below meant to capture a user's email address.

And the simple line of code written to create this element:

      
        <input class="f-email" formally="email" id="email" type="email">

The class attribute has several options for form validation and is optional. You'll notice in the above input that values are being judged against email address validity, with the input box turning green or red to signal success and failure of the value entered. There are two other optional class attributes that can be used for validity, and they are listed below:

      
        class="f-required"
        class="f-password"

f-required is used to make a field required, and validates on any input received. f-password validates only on a password containing at least one uppercase and lowercase letter, a number, and a special character ($@$!%*#?&). 

      
        <input class="f-password" formally="password" id="password" type="password">

The formally attribute is where you will want to define your label for the input you're creating. This should be a description of your form element and will also be used to construct the label that will make these forms accessible to all.

The id attribute is a required field on all form elements utilizing the Formally structure. The id will serve to link input elements to their respective labels and therefore tie all of the magic together.

The final attribute type is the HTML type attribute for all form elements. Let's take a look at some other types...

Foods I Enjoy

      
        <input formally="Cool Ranch" type="radio" name="doritos" value="ranch" checked>

You'll notice that these follow a very similar formula. I hope you'll also notice that the ID was omitted. As a fail-safe, Formally will create an ID for you in the case that it was omitted, but please do not forget to include the ID!

Buttons!

An optional formally attribute can be added to a select box to provide it with a label describing its contents. It will default to formatting much like the form labels rather than the header used for the button.
